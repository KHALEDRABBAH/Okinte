## 📖 Overview

**Okinte** is an enterprise-grade application tracking and document management platform. Designed with a strict focus on mobile performance, robust security, and seamless user experience, the system handles complex application workflows, document feedback loops, and secure administrative operations.

## ✨ Key Features

- **🚀 High-Performance Architecture:** Optimized for both mobile and desktop with dynamic component imports, ensuring a sub-2.5s Largest Contentful Paint (LCP) and perfect Lighthouse scores.
- **📁 Advanced Application Workflow:** Sophisticated handling of "Returned" applications, allowing users to efficiently upload replacement files and respond to feedback while preserving existing data.
- **🛡️ Enterprise-Grade Security:** Production-hardened infrastructure featuring strict asynchronous rate-limiting, XSS protection, brute-force prevention, and database-level admin role verification.
- **📧 Transactional Email System:** Fully configured, premium-branded automated email notifications backed by standard DNS deliverability protocols (DKIM, SPF, DMARC).
- **💳 Secure Checkout & Transactions:** Atomic tracking and transactional integrity for promotional codes and payment flows.
- **🗄️ Optimized Data Layer:** Highly efficient database operations featuring soft-delete aware queries and the elimination of N+1 bottlenecks.

## 🛠️ Tech Stack

### Frontend
- **Framework:** React / Next.js
- **Styling:** Tailwind CSS 

### Backend & Database
- **Runtime/API:** Node.js

## 🚀 Getting Started

### Prerequisites
Make sure you have the following installed:
- Node.js (v18+)
- PostgreSQL

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/KHALEDRABBAH/Okinte.git
   ```
2. Navigate to the project directory:
   ```bash
   cd Okinte
   ```
3. Install the dependencies:
   ```bash
   npm install
   ```
4. Set up your environment variables:
   Copy `.env.example` to `.env` and fill in your local database credentials and API keys.
   ```bash
   cp .env.example .env
   ```
5. Run database migrations:
   ```bash
   npx prisma migrate dev
   ```
6. Start the development server:
   ```bash
   npm run dev
   ```

## 🔒 Security & Best Practices

Okinte follows strict security guidelines:
- No insecure fallback environment variables.
- All administrative mutations are wrapped in database transactions.
- Sensitive routes are protected by server-side authentication guards.
